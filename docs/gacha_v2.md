# Гача v2 — спека

Статус: **зафиксированы решения по дизайну**, открыты числовые параметры
(см. блок «Открытое для балансировки»). Этот документ — единственный
источник истины при разработке; меняем его до кода, не наоборот.

## Стратегия выкатки (2026-06-16)

Решено: **одна ветка в проде, без параллельного `/gacha/v2`**. Переход
делаем **in-place** — старый `/gacha` эволюционирует в v2 (визуал v1,
который мы только что выкатили, остаётся как UI, докручиваем под новый
контракт по мере надобности). Точка отката — git-тег `gacha-v1-baseline`.

Порядок:
1. **Транзишн** in-place: gems-валюта, level/exp, mihoyo-pity, бой-движок,
   PvP-сервис — поверх существующих `gacha_collection`/`clicker_farm`.
2. **Тестовый период**: всё инструментируем телеметрией (см. ниже). Цель —
   собрать живые данные по дропам/гарантам/экономике/винрейтам.
3. **Сбор статистики** по завершении теста → балансировочные правки чисел
   из блока «Открытое для балансировки» (числа крутим, дизайн не трогаем).
4. **Полный вайп** через существующий `wipe_farm_sync` / `/farmwipe
   <here|all|chat_id> CONFIRM` — фермы+гача+пул+история сносятся, всё
   пересоздаётся лениво уже на отбалансированных параметрах.

### Телеметрия теста (что логируем для балансировки)
- **Кругты**: на каждый ролл — rarity, char_id, pity-счётчики на момент,
  сработал ли soft/hard pity, проигран ли 50/50, цена в gems. (таблица
  `gacha_roll_log` или поле note в `economy_tx`).
- **Экономика gems**: эмиссия (ежедневка, возврат 5★, покупка за cp) и
  сжигание (крутки) — через `economy_tx` kind'ы.
- **Бой/PvP**: исход, число раундов, состав команд, винрейт по char_id и
  по редкости (таблица `pvp_battle`: teams jsonb, winner, rounds, log).
- **Прогрессия**: распределение по звёздам/уровням на конец теста.

## Сводка решений

| решение | значение |

| Идентичность | **Гибрид**: герой = фермовый воркер + ККИ-карта |
| Формат боя | **Auto-battler** по раундам |
| Команда | **5 vs 5** с формацией (фронт/бэк) |
| Статы карты | HP / ATK / DEF / SPD + 1 special ability |
| Прогрессия | Звёзды (1–5, через дубли) + уровень (через опыт) |

| Pity | mihoyo: soft pity с ~75 круток, hard pity 90 (UR гарант); 10-pull SR+ гарант; рейт-ап 50/50 с гарантом следующего UR на баннер |
| PvP-триггер | Mini App: **матчмейкинг** (очередь внутри чата) + **точечная дуэль** на выбранного |

| Опыт уровня | Победы в боях + конвертация cp/gems в опыт |
| Связь с фермой | Звезда и уровень карты повышают её фермовый множитель (`farm_role`) и боевые статы (`card_role`) — обе оси растут на одних апгрейдах |

## Бой (auto-battler)

- Команды 5 vs 5 с формацией. Фронт (танки/брузеры) принимает удар первым,
  бэк (дамаги/саппорты) под защитой пока стоит хоть один фронт.
- Раунд: каждая выжившая карта по очереди делает действие (атаку или
  абилку), порядок — по SPD (выше → раньше). При равных SPD — случайно.
- Урон по простой формуле: `dmg = max(1, ATK_a − DEF_b * k)`, где `k` —
  балансировочный коэффициент (см. ниже). Special ability перекрывает /
  модифицирует действие раз в N раундов или по условию.
- Бой длится до уничтожения одной из команд или до hard-cap раундов
  (timeout → побеждает команда с большим суммарным HP%).
- Лог боя: компактный массив событий (`{round, actor, action, target, dmg}`)
  — UI разыгрывает анимацию по этому логу.

## Карта (UI и данные)

- На карте: art (`card_art`), имя, редкость (рамка по тиру), звёзды,
  уровень, 4 цифры статов, иконка/описание special.
- Рамку, имя, статы и звёзды **рисует UI программно** по редкости —
  художник даёт только `card_art` без оверлеев.
- Боевой токен в ряду 5-карт: маленький круг с `battle_token`,
  HP-полоса, индикатор готовности абилки.

## Прогрессия

- **Звёзды** 1→5 через дубли. Каждая звезда: +X% к статам и +Y% к
  фермовому множителю (`X`, `Y` — open params).
- **Уровень** 1→`MAX_LEVEL` через опыт. Уровень даёт линейный рост
  каждого стата по формуле тира редкости. Уровень-кап растёт с
  звёздами (5★ снимает потолок до `MAX_LEVEL`).
- **Опыт**:
  - победа в бою → пакет `exp_per_win` (величина зависит от типа боя),
  - конвертация: 1 cp = X exp / 1 gem = Y exp (см. open params).

## Дубли

- Дубль персонажа < 5★ → +1★, статы и фермовый множитель пересчитываются.
- Дубль 5★ → возврат gems по таблице редкости (open params), для
  закрытого цикла «крутка ↔ дубль».

## Валюта и крутки

- Единственная валюта крутки — `gems`.
- Покупка gems: за cp, по фикс-курсу (open param `CP_PER_GEM`). cp
  берётся из фермы или из гривен через AMM-рынок (обратный поток уже
  спроектирован в `market_service.buy_cp`).
- Одна крутка = `1 gem`, x10-крутка = `9 gems` (~стандартная скидка),
  цены в gems open.
- Pity-счётчики: `pity_ur_count` (до 90), `pity_sr_count` (до 10),
  `rate_up_lost` (флаг проигранной 50/50). На баннере один rate-up UR.

## Pity (mihoyo-style)

- Базовый дроп UR ~0.6% до круток ≥ `SOFT_PITY_START` (75).
- Линейный рост вероятности UR с 75 до 90 (`HARD_PITY = 90` —
  гарант UR).
- Каждая 10-я крутка с гарантом SR+ если выпал R.
- На выпадении UR: 50/50 — это баннерный UR или случайный из пула UR.
  Если случайный (lost rate-up) — следующий UR гарантированно баннерный
  (флаг сбрасывается).

## PvP

- **Матчмейк** (Mini App): жмёшь «найти бой» → попадаешь в очередь по
  чату (по умолчанию — ELO-окно). Как только нашёлся противник —
  автобой, лог проигрывается анимацией.
  - Награды: пакет gems + exp (open params), ELO-движение по
    стандартной формуле, ладдер чата.
- **Точечная дуэль**: вызываешь конкретного юзера (как нынешний `duel`),
  с кастомной ставкой в **гривнах**. Противник принимает в Mini App,
  бой запускается, победитель забирает обе ставки минус комиссия.

## Контракт ассетов для художника

Папка: `miniapp/static/gacha/<char_id>/`

| файл | размер | формат | где используется |
|---|---|---|---|
| `portrait_anim` | 512×512, alpha | webp (анимированный, loop 2–4с) | коллекция, баннер крутки, экран героя, фермовый слот |
| `card_art` | 768×1024 портретная, alpha | webp / png статика | лицевая сторона ККИ-карты (рамка/имя/статы — UI) |
| `battle_token` | 256×256, alpha | png (один спокойный кадр; опц. второй — атака) | боевой токен 5v5, обрамляется UI в круг |
| `splash_art` (опц.) | 1920×1080 или 1080×1920 | webp / jpg | reveal-экран при выпадении UR; не блокер |

**Гайдлайны:**
- Прозрачный фон везде где есть.
- `card_art` — центрированный портрет, оставить сверху место под имя,
  снизу — под полосу статов; рамка-overlay программная по редкости.
- `battle_token` — крупный план лица/верх торса, без длинных конечностей.
- `portrait_anim` — лёгкий цикл (дыхание/моргание), не отвлекающий.

## Открытое для балансировки (числа)

Не дизайнерские, а параметры; крутятся итерациями на живых данных:

- Стартовый каталог: количество персонажей по тирам R/SR/SSR/UR.
- Точные ставки дропа в банальном пуле и на rate-up банере.
- `SOFT_PITY_START` (75) / `HARD_PITY` (90) — стандартно, можно крутить.
- Цены: `gem` за cp, x1/x10 крутка, distill 5★ по редкости.
- Стат-формула: базовые стат-тиры, рост per-level, бонус per-star, `k`
  в формуле урона.
- `MAX_LEVEL`, кап уровня по звёздам.
- Пакет наград матчмейка (gems/exp/ELO-формула).
- Лимиты: круток в день нет; боёв в день — обсуждается (anti-spam).
- Длительность бойного timeout (раундов).
- Комиссия на ставку в точечной дуэли.

## Контуры реализации (план тракта)

1. **Каталог карт** (БД-модель + админ-импорт):
   - `cards`: char_id, name, rarity, base_stats, ability_id, asset_dir
   - `card_abilities`: id, name, description, effect_data jsonb
2. **Коллекция игрока** (расширение существующей `gacha_collection`):
   - level, exp, current_star, copies — поверх существующего.
3. **Бой-движок** (`bot/services/battle_service.py`):
   - `simulate_battle(team_a, team_b) → { winner, log }` — чистая функция,
     детерминированная по seed (для воспроизведения).
4. **PvP-сервис** (`bot/services/pvp_service.py`):
   - матчмейкинг-очередь (Redis), создание боя, выплата наград.
   - вызов точечной дуэли с ставкой (повторно использует часть `duel`).
5. **API**: `/api/v1/gacha/v2/*` (банер, ролл, коллекция, прокачка,
   бой, матчмейкинг, ладдер) — параллельно со старым `/gacha`,
   потом миграция.
6. **Mini App**: новые экраны (банер, ролл с reveal, коллекция-v2,
   карта-детали, команда, бой, матчмейк, ладдер).
7. **Dev-плейграунд** (`/dev/gacha`): превью контракта ассетов в 4-х
   реальных контекстах (карта, токен, портрет в коллекции, фермовый
   слот) — для художника, без бэкенда.

## Что НЕ в этой спеке (на будущее)

- Сезоны / ребейс ладдера.
- Бой PvE (волны/боссы) — отдельный трек, если решим делать.
- Колоды / pre-set формации (сейчас формация задаётся при поиске боя).
- Trading / market карт.
