
# 📘 **Xyloz Telegram Analytics Platform**

## 📊 О проекте

Это масштабируемая платформа для **сбора, хранения и анализа данных из Telegram-чатов**.
Архитектура состоит из микросервисов, использует очереди, модульные обработчики и гибкую схему данных, что позволяет:

* собирать сообщения, реакции, ответы и медиа (как ссылки);
* анализировать структуру разговоров и behavioral patterns;
* строить метаданные: частоты слов, активность пользователей, граф связанности;
* интегрировать ML/NLP модели для продвинутой аналитики;
* подключать веб-панель для визуализации.

Проект легко масштабируется и подходит как для небольших чатов, так и для больших каналов с нагрузкой 100k+ сообщений в сутки.

# 🔍 Основные компоненты

## **1) Telegram Bot (`bot/`)**

Отвечает за:

* приём сообщений
* сбор медиа (как ссылки/file_id)
* сбор данных о реакциях
* отправку событий в очередь Redis
* базовые команды (`/stats`, `/help`)

Использует **aiogram 3**.

---

## **2) Worker (`worker/`)**

Обрабатывает сообщения и реакции:

* извлекает текст, эмодзи, стикеры
* строит частотные словари
* обрабатывает цепочки reply/thread
* сохраняет всё в БД
* обновляет агрегированную статистику
* вызывает NLP-сервис (опционально)

Выполняет всё асинхронно и горизонтально масштабируется.

---

## **3) Backend API (`api/`)**

Сервис для:

* выдачи статистики
* поиска по сообщениям
* работы с реакциями
* подготовки графиков для dashboard
* получения структурированных данных по чатам

Реализован на **FastAPI**.

---

## **4) NLP / ML сервис (`nlp/`)**

Изолированный модуль с модельками:

* тональность (sentiment analysis)
* токсичность
* тематическое моделирование
* эмбеддинги
* кластеризация сообщений

Можно развернуть на GPU отдельно от остальных сервисов.

---

## **5) Dashboard (Web UI)**

Компоненты фронтенда:

* Активность пользователей
* Топ слов / эмодзи
* Структура диалогов (reply graph)
* Аналитика реакций
* Тональность сообщений
* Поиск по истории чата

Реализация: **React / Next.js** (или HTMX лёгкий вариант).

# 🧠 Краткий пересказ последних сообщений (LLM)

В боте доступна команда:

* `/summary` — пересказать последние 20 сообщений
* `/summary 50` — пересказать последние 50 сообщений
* `/sum 50` — короткий алиас
* `/summary_custom 100 | сфокусируйся на задачах и дедлайнах` — пересказ с кастомным фокусом
* `/sumc 50 | выдели только блокеры и решения` — короткий алиас кастомного режима
* `/prompt_show` — показать текущий prompt (только админ)
* `/prompt_set ...` — обновить prompt в БД (только админ)
* `/prompt_reset` — сбросить prompt к дефолту (только админ)
* `/model_show` — показать текущую модель пересказа (только админ)
* `/model_list` — показать список доступных моделей (только админ)
* `/model_set ...` — сменить модель пересказа без перезапуска (только админ)

# 📰 Дайджест чата (LLM)

* `/digest` — дайджест за последние 7 дней
* `/digest 14` — дайджест за произвольный период (1–30 дней)

Автоматический еженедельный дайджест отправляется каждый понедельник в 09:00 МСК во все чаты, где у бота была активность за последние 14 дней.

Ограничения:
* максимум `N` задается через `AI_MAX_MESSAGES` (по умолчанию `1000`)
* длинные сообщения обрезаются, а общий контекст ограничивается разумным токен-бюджетом перед отправкой в LLM
* лимиты можно настроить через `AI_MAX_INPUT_TOKENS` и `AI_MAX_CHARS_PER_MESSAGE`

Во время генерации бот обновляет одно и то же сообщение (динамический вывод), а не отправляет новые сообщения по шагам.

Для работы команды задайте переменные окружения:

* `YANDEX_API_KEY` — API ключ Yandex Cloud (обязательно)
* `YANDEX_FOLDER_ID` — ID каталога Yandex Cloud (обязательно)
* `YANDEX_MODEL` — модель по умолчанию (например `yandexgpt/latest`)
* `YANDEX_AVAILABLE_MODELS` — список моделей для `/model_list` через запятую
* `AI_MAX_MESSAGES` — верхняя граница для `N` в `/summary N` (опционально, по умолчанию `1000`)
* `AI_MAX_INPUT_TOKENS` — лимит входного контекста (опционально, по умолчанию `12000`)
* `AI_MAX_OUTPUT_TOKENS` — лимит выходных токенов ответа (опционально, по умолчанию `1200`)
* `AI_MAX_CHARS_PER_MESSAGE` — лимит символов на одно сообщение (опционально, по умолчанию `800`)
* `AI_MAX_CUSTOM_PROMPT_CHARS` — лимит длины кастомного запроса (опционально, по умолчанию `1200`)
* `AI_MAX_RETRIES` — число попыток при `429` (опционально, по умолчанию `2`)
* `AI_RETRY_DELAY_SEC` — базовая задержка между повторами в секундах (опционально, по умолчанию `1.5`)
* `BOT_ADMIN_IDS` — список Telegram ID админов через запятую (например `12345,67890`)

> Старые имена `OPENROUTER_MAX_*` и `YANDEX_MAX_OUTPUT_TOKENS` / `YANDEX_MAX_RETRIES` / `YANDEX_RETRY_DELAY_SEC` продолжают работать как fallback. При желании можно почистить `.env`.
